docker内加可执行文件并定时报废

打docker

我的思路是先创建个简易容器，然后把项目文件拷贝进容器，最后装环境。

1.写Dockerfile，主要注意python版本与项目环境一致即可

参考yolov8版本的。若存在报错，则注释那一行

2.在Dockerfile路径下，形成镜像

docker build 自己的镜像名:版本号 .

3.查阅镜像ID

docker images

4.运行镜像

x
docker run -itd -p 3330(端口占用可改):80 镜像ID /bin/bash

5.查看容器ID

xxxxxxxxxx
docker ps

6.运行容器

xxxxxxxxxx
docker exec -it 容器名(ID) /bin/bash

7.退出容器

xxxxxxxxxx
ctrl+p+q

8.从主机复制所有项目文件到容器内部

xxxxxxxxxx
docker cp 本地主机项目文件绝对路径 容器ID:容器内部绝对路径

9.再次运行容器，并设置可用所有GPU

x
 docker run --gpus all -itd -p 3330(端口占用可改):80 镜像ID /bin/bash

10.安装项目的anaconda环境

我的package地址如下

xxxxxxxxxx
/opt/conda/lib/python3.10/site-packages/

添加定时约束

对要执行的python脚本首行添加定时约束

xxxxxxxxxx
import datetime
expiration_date = datetime.date(2024, 4, 25) + datetime.timedelta(days=180)
current_date = datetime.date.today()


assert current_date < expiration_date
if current_date > expiration_date:
  exit()

生成可执行文件

对加好时间约束的python脚本生成可执行文件，参考https://blog.csdn.net/qq_38703529/article/details/117521655

x
pip install pyinstaller
pyinstaller -F 加好时间约束的python脚本

成功后显示可执行文件地址，直接输入文件地址即可运行可执行文件。若存在报错，则需要先改源码，再重新生成可执行文件。

容器内删除源码

最后，别忘记容器内删除源码。这样，用户仅通过执行可执行文件，难以逾越时间限制。
